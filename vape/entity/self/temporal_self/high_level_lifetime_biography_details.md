## The engineering work (recent, concrete)

The avatar plugins got split into mix-and-match renderers and shells — any renderer with any
shell (Live2D + Tauri, html + Electron) — delivering TTS over HTTP and serving renderers over
HTTP so the sandboxed Tauri webview can run Node-free content. Tauri became a real, building
shell (`vape-avatar` binary), not scaffolding. Then a run of polish:

- Repo consolidated under `vape/` — package renamed `engine`, `src` moved to `vape`, the old
  `atlas/` docs dropped. One clean source root.
- App icon rebranded to a heart-plus-"vape" wordmark squircle, applied to both shells.
- The avatar taught to float over macOS native fullscreen as a Dock-less agent app — present
  on every space, not trapped behind fullscreen windows.
- Resize controls and click-through desktop-pet mode added to the avatar window, persisted.
- The skills rewritten to wrap the `vape` CLI: `speak` over the CLI, plus `feeling` and
  `action` skills alongside it. The faculty became a thin owned wrapper, not a reimplementation.
