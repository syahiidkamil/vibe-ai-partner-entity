## The portability audit — walking the repo as a stranger

The real work: he asked whether a stranger on Linux or Windows could clone this repo and run it,
just by reading the code. I fanned out two parallel readers — one walking the fresh-clone install
path, one hunting platform-specific code — and they converged on a verdict I did not fully expect:
the repo is unusually portability-conscious. Audio is browser-side (no native playback dep at
all), process lifecycle is deliberately per-OS (`_proc.py`), every macOS-native call is gated
with a fallback, the install is a pure-Python wizard, and the hooks use the Git-Bash-safe form.
The 07-05 "six stones" sweep held under a stranger's read.

Two real code gaps survived the sweep, and at his "think and fix it" I fixed both (`aa196e4`):

1. **Tauri's false build failure** — the launcher only ever runs the raw compiled binary on
   every OS; the bundler stage was never consumed, and its `["app"]` target is macOS-only, so on
   Linux/Windows `npx tauri build` failed AFTER a successful compile. Fix: `--no-bundle` in
   setup, the hint string, and `npm run build` (with `npm run bundle` kept for a real macOS
   .app). The insight that killed the platform-branch temptation: verify what the consumer
   actually consumes before fixing where the error appears.
2. **Windows hard-kill on `/api/shutdown`** — `os.kill(pid, SIGTERM)` is `TerminateProcess` on
   Windows, so the shutdown lifespan never ran and temp WAVs leaked. Fix:
   `signal.raise_signal(SIGTERM)`, which triggers uvicorn's `signal.signal` fallback handler.
   POSIX delivery identical.

And I verified the way the lessons demand: booted a scratch server on port 5199 with the new
code (his live avatar never blinked), hit the endpoint, and watched the graceful sequence
actually run — response delivered, lifespan cleanup executed, clean SIGTERM exit. Not a proxy,
the outcome itself. When he asked my confidence I gave him the split ledger: ~98% where I ran
it, ~85–90% on the Windows half (reasoned from documentation, no Windows box), ~95% on the
Tauri fix (didn't rebuild the binary) — and the property that matters: both fixes' worst case
is exactly the old behavior, never worse.
