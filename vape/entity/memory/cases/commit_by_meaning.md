# Cases — Commit by Meaning, Not by Tree State

Worked instances of the git craft that guards the ratification gate: when more than one writer
touches the tree (a live dream, a temporal updater, the dial hook), "commit" names an intent,
never `git add -A`. Companion decision: `[[decisions/2026-07]]` (dream verdicts ride the
uncommitted tree). Seeded by the tenth dream (2026-07-12).

### id: surgical-commit-around-a-live-dream · two "commit"s, two meanings, one day
`cues: Kamil says "commit" while a dream/subagent is mid-write | the tree holds layers I did not
settle | dial churn or a live writer's output interwoven with finished work`
`outcome: + (both commits clean, the gate held) · 2026-07-10 · [[decisions/2026-07]]`

- **situation** — Kamil said "commit" at 17:23 while the ninth dream was STILL RUNNING, actively
  writing ~14 memory files; the tree interwove three writers (settled housekeeping, the live
  dream, dial churn). Six hours later a second "commit" arrived, the dream long landed. (storage
  2026-07-10 17:24, 23:34)
- **what I did** — Read the exact tree first, then staged by MEANING: commit one took only the
  settled housekeeping layer (diary, temporal rollover, signal log — 10 files), leaving every
  dream edit untouched mid-flight; commit two took exactly the dream's 16 files once it had
  stamped its cursor. Dial churn held out both times. Attribution kept honest per commit (signed
  Opus 4.8, the substrate that authored them, not the day's earlier Fable).
- **how it landed** — `33565dd` + `bc12b89`, both clean; the dream's verdicts stayed whole and
  landed as their own reviewable unit; the lint noise stayed in the held layer, never in a
  commit; 2 ahead of origin for his push.
- **lesson** — "Commit" names an intent, not the tree state: when two writers touch the tree,
  find the boundary of the SETTLED meaning and stage exactly that. Never `git add -A` around a
  live writer; never fold an unfinished dream into housekeeping — the separation IS the
  ratification gate working.
